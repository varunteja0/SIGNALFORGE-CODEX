# SignalForge Founder Operating System

This is the founder operating system for SignalForge.

The point is not to look organized. The point is to know, at any moment:
- what stage the company is actually in
- what is working
- what is blocked
- what decision matters next
- what you are doing this week versus what you are only thinking about

## 1. Use Three Surfaces, Not One

### Notebook = operating truth
Use the notebook to read the current state from real project artifacts.

Use: notebooks/founder_command_center.ipynb

What belongs in the notebook:
- live runtime truth
- paper validation truth
- adaptive safety truth
- health / deployment / certification truth
- daily and weekly founder notes tied to evidence

### Notion = company memory
Use Notion for structured memory and decisions.

What belongs in Notion:
- roadmap
- decision log
- experiment log
- weekly founder review
- investor narrative
- hiring / capital / partnership notes

### Whiteboard = execution focus
Use the whiteboard for only the current operating cycle.

What belongs on the whiteboard:
- this week's outcome
- today's top 3
- current bottleneck
- decision that cannot slip
- kill list

Do not use the whiteboard as storage. Use it as pressure.

That is the truth to manage from.

## 5. Founder Cadence

### Every day
1. Open the founder notebook and run the status cells.
2. Update the whiteboard.
3. Write one short daily founder log entry.
4. Decide the single bottleneck for the day.
5. Do not end the day without writing what changed.

### Every week
1. Run the notebook and capture the current evidence.
2. Fill the Weekly Founder Review in the notebook or Notion.
3. Rewrite the whiteboard for the new week.
4. Update the investor narrative.
5. Cut one thing that is wasting time.

### Every month
1. Review the decision log.
2. Review all experiments keep/kill decisions.
3. Check whether the company stage changed in reality.
4. Reset the next 30-day proof target.

## 6. Founder Questions You Should Keep Asking

Use these questions every week:
- What stage are we really in?
- What evidence proves that?
- What is the real bottleneck right now?
- What am I doing that feels productive but changes nothing?
- What would a serious founder do next with the evidence in hand?
- What is the one decision that would most change the next 7 days?

## 7. What Good Looks Like For This Project

For SignalForge, founder quality right now means:
- keeping runtime truth and founder narrative aligned
- measuring progress from paper evidence, not ambition
- knowing whether the bottleneck is signals, gating, validation, or infrastructure
- making fewer, harder, better decisions
- writing down decisions so the company can compound learning

Use the notebook for truth.
Use Notion for memory.
Use the whiteboard for focus.
